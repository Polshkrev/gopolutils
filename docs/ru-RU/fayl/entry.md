# Запись
Стандартизация записи в файловой системе. Этот тип предназначен для упрощения операций, относящихся к конкретному файлу, а не просто к пути.

### Использование
### Конструкция
Для создания новой записи определен конструктор `NewEntry`, который принимает [Адрес](/docs/ru-RU/fayl/path.md) и возвращает указатель на новую запись `Entry`, тип которой установлен на `File`, а содержимое представляет собой пустой инициализированный срез.
### Функциональность
Каждое из свойств записей имеет понятные методы получения и установки значения. Чтоб созданить файла определены три метода:
- `Touch`: Чтоб созданить записи типа `File`.
- `MakeDirectory`: Чтоб созданить записи типа `Directory`.
- `Create`: Универсальная обертка над вышеупомянутыми методами, которая вызывается в зависимости от внутреннего `EntryType` записи.

Каждый из вышеупомянутых методов может завершиться ошибкой, если путь к записи уже существует или если запись не соответствует указанному типу.

Чтобы удалить записи существует очень похожий набор методов:
- `RemoveFile`: Удаление записи типа `File`.
- `RemoveDirectory`: Создание записи типа `Directory`.
- `Remove`: Универсальная обертка над вышеупомянутыми методами, которая вызывается в зависимости от внутреннего `EntryType` записи.

Каждый из вышеупомянутых методов может завершиться ошибкой, если путь к записи не существует, если запись не соответствует указанному типу или если тип записи не может быть определен.

Методы создания и удаления реализуются одновременно и используют безопасные для параллельного выполнения сборки мусора.
