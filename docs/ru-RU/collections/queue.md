# Очередь

### Добавить
Чтобы добавить в очередь есть два метода можно использовать; есть &mdash; конечно же &mdash; метод `Append`. Этот метод добавляет один элемент в очередь. Есть тоже метод `Extend`. Этот метод добавляет несколько элементов в очередь в форме `View`.

### Удалить
Чтобы удалить из очереди существует метод `Dequeue`. Метод `Remove` определён, но его следует использовать редко. Как следует из названия, метод `Dequeue` возвращает первый добавленный элемент и удаляет его из очереди. Если очередь пуста, возвращается `ValueError` с нулевым указателем данных.

### Доступить
Чтобы доступить к элементу в очереди есть &mdash; как обсуждалось ранее &mdash; **деструктивный** метод `Dequeue` и **недеструктивный** метод `Peek`. Метод `Peek` возвращает первый добавленный элемент, но не удаляет его из очереди. Если очередь пуста, Возвращается `ValueError` с нулевым указателем на данные. Для обновления массива доступен метод `Update`. Этот метод принимает индекс, по которому находится обновляемый элемент, и новое значение, которое необходимо присвоить данному индексу. Этот метод либо обновит значение по указанному индексу и вернет нулевой указатель на исключение, либо вернет указатель на `ValueError`, если очередь пуста, или `OutOfRangeError`, если индекс превышает размер очереди.

### Реализовать
Эта структура реализует интерфейсы `Collection`, `View` и `Sized`.
